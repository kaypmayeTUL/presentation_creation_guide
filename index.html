HTML Presentation Authoring Guide

Kay P. Maye · Howard-Tilton Memorial Library

Build Library Instruction Presentations Without AI

A complete reference for creating, editing, and publishing the interactive HTML presentation format developed for Scholarly Engagement workshops at Tulane University. Copy-paste code included throughout.

01Introduction

These presentations are self-contained single HTML files — one file, no dependencies beyond Google Fonts. They require no frameworks, no build tools, and no server-side code. A plain text editor and a web browser are all you need.

Each file can be opened locally, emailed, or hosted on GitHub Pages. The format was designed for library instruction at Howard-Tilton Memorial Library. Core principles:

  • Self-contained: One file. All CSS, JavaScript, and content live inside it.
  • Accessible: Keyboard navigation (← → arrows), dropdown slide selector, touch/swipe support.
  • Interactive: Text inputs, quizzes, checklists, and Boolean builders embedded in slides.
  • Trackable: An activity badge counts completions and stays visible in the nav bar.
  • Exportable: A floating panel lets students email or download their activity responses.
  • Branded: Tulane green #285C4D, sky blue #71C5E8, and Google Fonts throughout.
💡
Recommended Editor

Use Visual Studio Code (free). Install the Prettier extension for auto-formatting and Live Server to preview changes instantly in your browser as you type. For quick edits, Notepad++ on Windows or TextEdit (in plain-text mode) on Mac also work.

Every presentation follows this structure inside a single .html file. The five pieces below appear in roughly the same order in every file you make.

  </style>

    /* ⑤ JavaScript */
    // Navigation logic (showSlide, totalSlides)

    // Quiz / checklist logic
    // Export (email / download)
  </script>

</body>
</html>

<!-- ============================================================ JAVASCRIPT -->
<script>
/* ── Configuration ─────────────────────────────────────────── */
const totalSlides    = 3;   // ← UPDATE THIS when you add/remove slides
const totalActivities = 2;  // ← UPDATE THIS to match activity count
const presTitle      = "YOUR PRESENTATION TITLE";
const contactEmail   = "kmaye@tulane.edu";

/* ── State ──────────────────────────────────────────────────── */
let currentSlide = 1;
let activitiesCompleted = new Set();

/* ── Navigation ─────────────────────────────────────────────── */
function showSlide(n) {
  document.querySelectorAll('.slide').forEach(s => s.classList.remove('active'));
  document.getElementById('slide-' + n).classList.add('active');
  currentSlide = n;
  document.getElementById('slide-counter').textContent = n + ' / ' + totalSlides;
  document.getElementById('slide-select').value = n;
  document.getElementById('prev-btn').disabled = (n === 1);
  document.getElementById('next-btn').disabled = (n === totalSlides);
  window.scrollTo(0, 0);
}
function nextSlide() { if (currentSlide < totalSlides) showSlide(currentSlide + 1); }
function prevSlide() { if (currentSlide > 1) showSlide(currentSlide - 1); }

/* Keyboard + swipe */
document.addEventListener('keydown', e => {
  if (e.key === 'ArrowRight' || e.key === 'ArrowDown')  nextSlide();
  if (e.key === 'ArrowLeft'  || e.key === 'ArrowUp')    prevSlide();
});
let touchStartX = 0;
document.addEventListener('touchstart', e => { touchStartX = e.touches[0].clientX; });
document.addEventListener('touchend',   e => {
  const dx = e.changedTouches[0].clientX - touchStartX;
  if (dx < -50) nextSlide();
  if (dx > 50)  prevSlide();
});

/* ── Activity Tracking ─────────────────────────────────────── */
function saveActivity(num) {
  activitiesCompleted.add(num);
  document.getElementById('act' + num + '-confirm').style.display = 'block';
  updateBadge();
}
function updateBadge() {
  document.getElementById('activity-badge').textContent =
    activitiesCompleted.size + ' / ' + totalActivities + ' activities';
}

/* ── Export ─────────────────────────────────────────────────── */
function toggleExport() {
  const body  = document.getElementById('export-body');
  const arrow = document.getElementById('export-arrow');
  const open  = body.classList.toggle('open');
  arrow.textContent = open ? '▼' : '▲';
}
function collectResponses() {
  // Collect all textarea values — add more IDs as you add activities
  const ids = ['act1-response', 'act2-response'];
  let out = presTitle + ' — Activity Responses\n';
  out += new Date().toLocaleDateString() + '\n';
  out += '═'.repeat(50) + '\n\n';
  ids.forEach((id, i) => {
    const el = document.getElementById(id);
    if (el && el.value.trim()) {
      out += 'Activity ' + (i + 1) + ':\n' + el.value.trim() + '\n\n';
    }
  });
  return out;
}
function emailResponses() {
  const body = encodeURIComponent(collectResponses());
  window.location.href = 'mailto:' + contactEmail + '?subject=' +
    encodeURIComponent(presTitle + ' Responses') + '&body=' + body;
}
function downloadResponses() {
  const blob = new Blob([collectResponses()], { type: 'text/plain' });
  const a = document.createElement('a');
  a.href = URL.createObjectURL(blob);
  a.download = 'responses.txt';
  a.click();
}
</script>
</body>
</html>

18Scored Quiz Pattern

Used in The Search Ecosystem (6-question Boolean quiz) and the Python / R presentations. Each question is an object in a JavaScript array.

HTML + JavaScript — self-contained scored quiz 
<!-- Place this HTML inside any slide -->
<div id="quiz-container"></div>
<div id="quiz-results" style="display:none; padding:16px; background:var(--green-light);
     border-radius:8px; margin-top:12px;"></div>

<!-- Place this JavaScript in the <script> block -->
<script>
const quizQuestions = [
  {
    question: "Which operator narrows a search by requiring BOTH terms to appear?",
    options: ["OR", "NOT", "AND", "NEAR"],
    correct: 2,  // 0-indexed: "AND" is index 2
    explanation: "AND narrows results — both terms must be present."
  },
  {
    question: "Which operator broadens a search to include EITHER term?",
    options: ["AND", "OR", "NOT", "WITH"],
    correct: 1,
    explanation: "OR broadens results — either or both terms may appear."
  },
  // ← Add more question objects here
];

let quizAnswers = {};

function renderQuiz() {
  const container = document.getElementById('quiz-container');
  container.innerHTML = quizQuestions.map((q, i) => `
    <div style="background:white; border:1px solid var(--border); border-radius:8px;
                padding:14px 16px; margin-bottom:12px;">
      <p style="font-weight:600; font-size:0.92rem; margin-bottom:10px;">
        ${i+1}. ${q.question}
      </p>
      ${q.options.map((opt, j) => `
        <label style="display:flex; align-items:center; gap:8px; margin-bottom:6px;
                      cursor:pointer; font-size:0.88rem;">
          <input type="radio" name="q${i}" value="${j}"
                 onchange="quizAnswers[${i}]=${j}">
          ${opt}
        </label>
      `).join('')}
    </div>
  `).join('');

  container.innerHTML += `
    <button class="save-btn" onclick="scoreQuiz()">Submit Quiz</button>`;
}

function scoreQuiz() {
  let score = 0;
  let feedback = '';
  quizQuestions.forEach((q, i) => {
    const chosen = quizAnswers[i];
    const correct = chosen === q.correct;
    if (correct) score++;
    feedback += `<p style="margin-bottom:8px;">
      <strong>Q${i+1}:</strong>
      ${correct ? '✅' : '❌'}
      ${!correct ? '<em>(' + q.options[q.correct] + ')</em> — ' : ''}
      ${q.explanation}
    </p>`;
  });
  const results = document.getElementById('quiz-results');
  results.style.display = 'block';
  results.innerHTML = `
    <h3 style="font-family:var(--font-heading); color:var(--green); margin-bottom:10px;">
      Score: ${score} / ${quizQuestions.length}
    </h3>
    ${feedback}`;
  saveActivity(5); // mark quiz as completed — use the right activity number
}

// Call renderQuiz() when the slide loads, or on page load:
document.addEventListener('DOMContentLoaded', renderQuiz);
</script>

22Troubleshooting

Clicking Next/Prev goes past the last or first slide

totalSlides in the JavaScript is wrong. Count your <div class="slide"> elements and set totalSlides to that number.

Dropdown shows wrong slide names or skips a slide

A <option value="N"> is missing or has a wrong value. The value attribute must exactly equal the slide number in the id="slide-N".

Activity badge doesn't increment

The onclick="saveActivity(N)" call uses a number that doesn't match or the function isn't present. Also check that actN-confirm exists as a paragraph id.

Two slides show at once / first slide bleeds into the second

Almost always a malformed attribute or missing closing tag in the slide above. Common culprits: <p class ="presenter> (missing closing quote, extra space before =), or a </div> that was deleted when editing. Validate the HTML in any online HTML validator if you can't spot it.

Google Fonts aren't loading (offline / internal network)

The presentations load fonts from fonts.googleapis.com. If the computer is offline or on a restricted network, add a CSS fallback at the end of each font-family declaration, e.g., font-family: 'Source Serif 4', Georgia, serif;. The presentations already include these fallbacks via the CSS variables.

Export email opens but body is blank

The student hasn't saved any activities yet (nothing to export). If you want to test it, type into a textarea and click Save first.

Slide content is cut off / not scrolling

The slide div needs overflow-y: auto; in its CSS. Check that the .slide rule includes this property. If a single slide has too much content, consider splitting it into two slides.

Special characters (&, <, >) display as garbled text

In HTML content, these characters must be escaped:

  • & → &amp;
  • < → &lt;
  • > → &gt;

In JavaScript strings (inside <script>), use the raw characters.
